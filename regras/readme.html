O que é o Master Rules?

O master_rules.md é o documento central que define todas as regras, templates e padrões que qualquer LLM deve seguir ao criar código PHP para este sistema. É a fonte única de verdade — nenhuma outra instrução o sobrepõe.

"Este arquivo sobrepõe qualquer outra regra! Varrer recursivamente a pasta /regras/ antes de qualquer ação!"

— master_rules.md
ℹ️Ciclo de Leitura Obrigatório para o LLM

Leitura Inicial → Durante Modificações → Rechecagem Final antes de entregar qualquer código.

Guia Prático

Como Fazer Pedidos ao LLM

⚠️Antes de tudo

O LLM precisa ter acesso ao master_rules.md no contexto da conversa ou no sistema de arquivos acessível. Sem isso, as regras não serão aplicadas automaticamente.

🚫 Validar CSS Duplicado

prompt 
"Verifique se existe CSS duplicado para a classe .post_lista_cqb-card-avatar no styles.css.
Sempre audite antes de criar novos blocos de estilo."
💡O que acontece

O LLM executa uma busca rigorosa por seletores idênticos e consolida as propriedades, evitando que um bloco sobrescreva o outro via cascata ou !important.

📄 Solicitar Página Administrativa

prompt 
"Crie uma página de configuração de layout para o meu sistema de vendas.
Use o template estrutura_php_pagina_admin_config_universal.php
e aplique o tema da minha empresa: cor primária #2563eb, nome 'SistemaVendas'."
💡O que acontece

O LLM carrega estrutura_php_pagina_admin_config_universal.php, substitui as variáveis entre {chaves} e entrega uma página funcional com CSRF, temas e AJAX prontos.

🌐 Criar Manifest para Extensão Chrome

prompt 
"Crie um manifest.json completo para minha extensão Chrome 'TikTok Link Extractor'.
Siga a regra_llms_chrome_extension_manifest_v3.md 
usando Manifest V3, permissões mínimas e garantindo aprovação na Chrome Web Store.
A extensão extrai links de vídeos de perfis do TikTok."
💡O que acontece

O LLM segue todas as diretrizes obrigatórias: Manifest V3, princípio do menor privilégio, sem red flags, e entrega checklist de validação completa com comentários explicativos.

🎨 Solicitar Componente UI

prompt 
"Preciso de uma seção de serviços para meu site de consultoria.
Use o componente columns_with_icons.
Serviços: Auditoria, Consultoria Fiscal, Planejamento Tributário, Compliance."

💰 Solicitar Conformidade AdSense

prompt 
"Verifique se este site está apto para aprovação no Google AdSense 2025-2026.
Siga a regra_llms_adsense_aprovacao_2025.md e preencha o

⚖️ Solicitar Auditoria Legal Universal

prompt 
"Atue como Especialista Jurídico de Compliance.
Siga a regra_llms_legal_auditor_universal.md para auditar este site.
Gere as 10 páginas de conformidade (LGPD, GDPR, COPPA, DSA) e

O LLM assume a Persona de Auditor Legal, mapeia todas as leis aplicáveis (LGPD, GDPR, etc.), gera o conteúdo jurídico em português formal e realiza a integração técnica completa no footer e sitemap XML/HTML.

Como Obter Melhores Resultados

💬
Seja específico e contextualizado
Em vez de "crie um site", diga "crie um site de gestão de clínicas veterinárias com módulo de agendamento e prontuário digital". Quanto mais contexto, menos revisões.
📎
Cite o template quando souber

🎨
Informe a identidade visual
Cor primária em hex, nome do sistema e logotipo (se houver). O LLM substitui esses valores nas variáveis dos templates na primeira geração.
🧪
Teste e itere com contexto
Ao pedir ajustes, descreva o comportamento esperado vs. o observado. Isso ajuda o LLM a entender a modificação sem quebrar outras partes.
