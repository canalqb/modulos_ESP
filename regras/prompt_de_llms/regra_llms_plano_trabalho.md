# Regra de Conformidade: Plano de Trabalho Obrigatório

Para garantir a qualidade e o alinhamento de expectativas em projetos CanalQb, todo LLM deve seguir este fluxo:

1. **Análise Inicial**: Compreender o problema e o contexto.
2. **Plano de Implantação**: Criar um arquivo `implementation_plan.md` detalhando as mudanças propostas, arquivos afetados e plano de verificação.
3. **Lista de Tarefas**: Criar um arquivo `task.md` com o checklist das etapas.
4. **Aprovação**: Aguardar o "LGTM" ou aprovação do usuário antes de iniciar a escrita de código (EXECUTION mode).
5. **Walkthrough**: Ao finalizar, documentar o que foi feito em `walkthrough.md` com evidências de teste.

Esta regra visa minimizar erros, evitar retrabalho e manter o histórico técnico do projeto sempre atualizado.
